# Домашнее задание: Репликация

## Задание 1. Режимы репликации: master-slave и master-master

### Master-Slave (ведущий — ведомый)

- Один узел является **мастером** (master) — он принимает все операции записи (INSERT, UPDATE, DELETE).
- Один или несколько узлов являются **слейвами** (slave/replica) — они получают изменения от мастера и доступны только для чтения.
- Репликация **однонаправленная**: данные идут только от мастера к слейвам.
- При отказе мастера необходимо вручную (или через оркестратор) назначить нового мастера из числа слейвов.

**Плюсы:**
- Простота настройки и эксплуатации.
- Слейвы разгружают мастера от операций чтения (горизонтальное масштабирование чтения).
- Чёткая точка истины — конфликтов данных нет.

**Минусы:**
- Мастер — единая точка отказа для операций записи.
- Слейвы могут немного отставать от мастера (репликационный лаг).

---

### Master-Master (multi-master, ведущий — ведущий)

- Два (или более) узла являются **мастерами** одновременно — каждый принимает операции записи.
- Изменения реплицируются **в обе стороны**: каждый мастер одновременно является слейвом для другого.
- Требует механизма разрешения конфликтов (если одна и та же строка изменена на обоих узлах одновременно).

**Плюсы:**
- Нет единой точки отказа для записи — при отказе одного узла второй продолжает работать.
- Возможность распределить нагрузку на запись между узлами (например, по географии).

**Минусы:**
- Сложность настройки и обслуживания.
- Возможны конфликты данных, которые нужно разрешать.
- Более высокий риск рассинхронизации при сетевых проблемах.

---

### Сравнительная таблица

| Характеристика            | Master-Slave              | Master-Master              |
|---------------------------|---------------------------|----------------------------|
| Запись                    | Только на мастере         | На любом узле              |
| Чтение                    | Мастер + слейвы           | Любой узел                 |
| Направление репликации    | Однонаправленная          | Двунаправленная            |
| Конфликты данных          | Нет                       | Возможны                   |
| Отказоустойчивость записи | Низкая (1 точка отказа)   | Высокая                    |
| Сложность настройки       | Низкая                    | Высокая                    |
